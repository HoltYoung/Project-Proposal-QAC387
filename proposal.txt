3. Problem Statement & Motivation 
Data analysis is a manual and repetitive process. An analyst looks at a dataset, comes 
up with a question, writes code, reads the results, and then figures out what to look at 
next. The quality of the analysis depends on the analyst's experience, and many people 
who need data insights lack the coding or statistics background to do it well. 
Existing LLM tools like ChatGPT's Code Interpreter can answer one-off questions about 
data, but they don't follow up on what they find. You get one answer and have to figure 
out the next question yourself. We want to build an LLM-based agent that wraps the 
scientific method loop (hypothesize, test, learn, repeat) around a dataset so it can 
conduct a multi-step analysis on its own. 
4. Application Concept & Features 
HypothesisLoop is a Python application where an LLM iteratively analyzes a dataset. 
The user provides a CSV and a research question, and the agent enters a loop: 
1. Hypothesize - The LLM proposes a testable hypothesis about the data 
2. Experiment - The LLM writes and runs Python code to test it 
3. Evaluate - The LLM interprets the results (p-values, effect sizes, etc.) 
4. Learn - Results are stored in a memory/trace system 
5. Repeat - The LLM uses what it learned to propose a new hypothesis 
After a set number of iterations (default 5), the agent produces a summary report with 
its findings, charts, and reasoning chain.

LLM Usage 
Our primary model will be Anthropic Claude Opus 4.6. The app will be built so users can 
swap in OpenAI or other API keys if they prefer. The LLM handles hypothesis 
generation (structured JSON output), code writing and execution, result interpretation, 
and report writing. We will pass the full history of previous iterations as context so the 
LLM can build on what it has already learned.

6. Data Source & Preprocessing 
Our primary dataset is the UCI Adult Income Dataset (publicly available, 48,842 
records, 14 attributes). It includes census data with demographic features like age, 
education, occupation, race, and sex, predicting whether income exceeds $50K/year. 
We chose it because it has both numerical and categorical variables and well-known 
relationships (e.g., education and income) we can use to check whether the agent finds 
real patterns. 
We will also test on the Kaggle House Prices dataset (79 features, regression) and the 
WHO Life Expectancy dataset (health and economic indicators) to see how well the 
agent generalizes. 
The agent will handle preprocessing (missing values, encoding, outlier detection) as 
part of its first iteration. We will compare its cleaning against a manual baseline.

8. Ethical Considerations, Bias Mitigation, & Safety 
‚ÄĘ‚Äč
Bias: The LLM could make unwarranted causal claims about sensitive variables 
(e.g., race and income). As a first step, the system prompt will instruct the model 
to distinguish correlation from causation. On top of that, we will add a 
post-processing check that flags causal language (words like "causes," "leads 
to," "due to") in any finding that involves a sensitive variable. Any flagged output 
will get a disclaimer header. We recognize the system prompt alone is not a 
reliable guardrail, so these layered checks are important. 

‚ÄĘ‚Äč
Overconfidence: Findings will include significance levels and caveats where 
appropriate. 
‚ÄĘ‚Äč
Data privacy: User data is sent to an external LLM API. We will notify users and 
suggest local alternatives for sensitive data. 
‚ÄĘ‚Äč
Transparency: Langfuse traces let users audit every step of the agent's 
reasoning. 
‚ÄĘ‚Äč
Safety: Sandboxed code execution, restricted imports, output size limits, and 
iteration caps. 
9. Expected Challenges 
‚ÄĘ‚Äč
Code bugs: LLM-generated code sometimes fails. We will add retry logic so the 
LLM can see and fix its own errors. 
‚ÄĘ‚Äč
Repetitive hypotheses: Later iterations may rehash earlier ones. We will track 
what has been tested and prompt for novelty. 
‚ÄĘ‚Äč
Context window limits: Long trace histories may not fit. We will summarize 
older iterations. 
‚ÄĘ‚Äč
Varying outputs: Results change between runs. We will run multiple times and 
compare.
